# Matt Pocock 技能包 — 小白用法

技能已经装在你电脑的 `C:\Users\LunaFolia\.cursor\skills\` 里。  
本仓库（shiloku）的配套配置在 `docs/agents/` 和 `AGENTS.md`。

## 你怎么用（就记三句话）

1. **在 Cursor 对话框里输入 `/` 再加技能名**，等于「请 AI 按这套流程干活」。
2. **忘了有哪些命令？** 输入 `/help`，会列出全部技能和用法。
3. **不知道用哪个？** 输入 `/ask-matt`，用大白话描述你想做什么，它会推荐技能。
4. **日常改网站**，直接像现在这样跟我说就行；只有想「正经做一功能」时才用下面那些技能。

## 最常用的几个

| 输入 | 干什么 | 什么时候用 |
| --- | --- | --- |
| `/help` | 列出所有技能和用法 | 忘了命令、想查说明书 |
| `/ask-matt` | 帮你选技能 | 不知道从哪开始 |
| `/grill-me` | 追问你把需求说清楚 | 想法很模糊时 |
| `/grill-with-docs` | 同上，并更新 CONTEXT.md | 改 shiloku 代码、功能较大时 |
| `/tdd` | 先写测试再写功能 | 加逻辑、怕改坏时 |
| `/diagnosing-bugs` | 按步骤查 bug | 出了问题查原因时 |
| `/handoff` | 把当前对话整理成交接文档 | 聊太长了要开新对话时 |

## 完整流程（可选，做大功能时）

```
/grill-with-docs  →  /to-prd  →  /to-issues  →  每个任务开新对话用 /implement
```

任务会写在 `.scratch/` 文件夹里，不用 GitHub。

## 示例

**例 1：想加「分享当前歌曲」按钮，但还没想好细节**

```
/grill-me 我想在音乐室加一个分享按钮，帮我把需求问清楚
```

**例 2：歌词又错位了**

```
/diagnosing-bugs 手机端歌词和播放器重叠了
```

**例 3：不知道选哪个**

```
/ask-matt 我想优化手机端页面，该用哪个技能？
```

**例 4：忘了有哪些命令**

```
/help
```

或带具体问题：

```
/help 我想改歌词滚动，该用哪个技能？
```

## 注意

- 技能名必须带 `/`，在 **Agent 模式** 的对话里输入。
- 第一次在本仓库用工程类技能前，配置已完成，无需再跑 `/setup-matt-pocock-skills`。
- 改 `CONTEXT.md` 或 `docs/agents/` 里的文件，AI 会越用越懂你的项目。
